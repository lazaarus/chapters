Data & Information

Data refers to raw facts and figures without context. Information is processed data that is meaningful and useful. Features of information include accuracy, relevance, timeliness, and completeness.

Database Management System (DBMS)

A DBMS is software that manages databases, ensuring data is stored, retrieved, and manipulated efficiently. It provides a systematic way of managing data and allows multiple users to access the database concurrently.

Objectives

  • Data Integrity
  • Data Security
  • Data Consistency
  • Data Redundancy Control

Advantages

  • Efficient Data Management
  • Data Security
  • Backup and Recovery
  • Concurrent Access

Disadvantages

  • Complexity
  • Cost
  • Performance Overhead

Types of Database Models

Hierarchical Model

Data is organized in a tree-like structure. Each child record has only one parent. Example: IBM's Information Management System (IMS).

Network Model

Data is organized using a graph structure, allowing multiple parent-child relationships. Example: Integrated Data Store (IDS).

Relational Model

Data is organized into tables (relations) with rows and columns. Example: MySQL, PostgreSQL.

Entity-Relationship Model

Data is represented using entities and relationships. Used for database design.

Integrity Constraints

Domain Integrity

Ensures that all values in a column are of the same type and within a valid range.

Entity Integrity

Ensures that each table has a primary key and that the key is unique and not null.

Referential Integrity

Ensures that foreign keys correctly point to primary keys in other tables.

Normalization

Normalization is the process of organizing data to reduce redundancy and improve data integrity.

Normal Forms

  • 1NF: Eliminate duplicate columns; create separate tables for related data.
  • 2NF: Remove partial dependencies; all non-key attributes must depend on the whole primary key.
  • 3NF: Remove transitive dependencies; non-key attributes must not depend on other non-key attributes.

Advantages

  • Eliminates Redundancy
  • Improves Data Integrity
  • Efficient Data Updates

Disadvantages

  • Complex Queries
  • Performance Overhead

Database Security

Database security involves measures to protect data from unauthorized access and corruption.

Challenges

  • Unauthorized Access
  • Data Breaches
  • Data Corruption

Security Measures

  • Access Control
  • Encryption
  • Auditing
  • Backup and Recovery

SQL Data Types

SQL provides various data types to define the nature of data in a table.

Character Types

  • CHAR: Fixed-length character string.
  • VARCHAR: Variable-length character string.

Binary Types

  • BINARY: Fixed-length binary data.
  • VARBINARY: Variable-length binary data.

Text Types

  • TINYTEXT: Very short text string.
  • TEXT: Short text string.
  • LONGTEXT: Long text string.

Numeric Types

  • INT: Integer.
  • FLOAT: Floating-point number.
  • DOUBLE: Double precision floating-point number.
  • DECIMAL: Fixed-point number.

Date and Time Types

  • DATE: Date value.
  • DATETIME: Date and time value.

Other Types

  • ENUM: Enumeration; a list of predefined values.
  • BIT: Bit-field.
  • BOOLEAN: Boolean value (TRUE or FALSE).
  • TINYINT: Very small integer.
